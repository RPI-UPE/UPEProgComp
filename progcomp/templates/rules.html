General

The programming competition will last 2 hours and will consist of four problems. Please remember our simple rules:

  1. All submitted source code must be your own.
  2. You are not allowed to communicate with/ask questions of any other person.
  3. You are allowed to use the internet as a reference.

Any attempt to tamper with the competition server is considered dishonesty with respect to the contest. Please remember you are always subject to RPI's COMEC policy.

Problems

All problems can be viewed when the competition begins, and there is no particular order in which the problems must be attempted. Each problem has a description, detailed input and output format, and a sample input and the respective expected output.

Please note that with regard to every input file, the first line will always contain the number of individual input cases that will be read from the file. Each input case follows the format specified in the problem description.

Submissions

Each submission consists of downloading a randomized test input file from our server, running it through your program, and then uploading the output of your program to the server. When you request an input file a two minute timer will be started. You will need to upload the output from your solution before the timer runs out for your solution to be accepted. If time does run out, then you will be given a new input file to use. A single source code file is also required so that the contest administrators may see the contents of your program for validation purposes only. Your source code will not impact your final standing.

If your output matches our expected output then your program is considered to have solved the problem. If it doesn't match up, we'll show you a small diff that can help you find what went wrong. An unlimited number of submissions may be made for each problem, and the number of submissions will not affect your standing in the scoreboard.

Winners

Winners will be decided based on first the number of problems successfully completed, and in case of a tie the winner will be whoever completed their problems first. The ranking for each contestant can be seen on the scoreboard page once the contest is underway.
